HTML tags and elements

A web page (or HTML page) is made up of content - the words you write and images you link - and HTML elements.

When a web browser loads a web page, it needs to understand what the contents of the page are and how to display them. We use HTML tags to define and differentiate our content.

For example, we might have two groups of words:

This text should be a headline. This bit of text is the first sentence of the story.

Without defining - or marking these up - these with tags, there's no meaning or structure.

If we mark them up with HTML tags to define what they are meant to be e.g. a heading and a paragraph, we start to see some structure:

<h1>This text should be a headline </h1>

<p>This bit of text is the first sentence of the story.</p>

Will display as:

This text should be a headline

This bit of text is the first sentence of the story.

It's important to distinguish between html tags and elements.

In the image below shows a h1 heading element. This is made up of:

  • an opening h1 tag
  • text content

The whole object forms an HTML element.

Structure of a HTML page

Hover your mouse over each tag to see a description.

<!DOCTYPE html>

<html lang="en">

<head>

<meta charset="utf-8">
<meta name="author" content="Hadrian Cawthorne">
<meta name="viewport" content="width=device-width, initial-scale=1">

<title>Web and Multimedia Design | the structure of a web page's HTML</title>

<link rel="stylesheet" href="css/custom.css">

</head>

<body>

<h1>Here is some text that is a header</h1>

<p>Here's some paragraph text.</p>

</body>

</html>

Once you have the basic structure of your HTML file setup, you can just work between the opening and closing body tags!
